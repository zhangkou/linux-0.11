## 1.1 Linux 0.11 是什么

Linux 0.11 是 Linus Torvalds 于 **1991年12月** 发布的早期 Linux 内核版本，也是第一个被广泛使用的稳定版本。

**基本特征：**
- 约 **10,000 行** C代码 + 汇编代码
- 运行在 **x86 (i386)** 架构
- 支持 **Minix文件系统**
- 最多支持 **64个进程**
- 最大支持 **16 MB 物理内存**
- 单用户、多任务操作系统

**历史意义：**
```
1991年8月  Linus 发布 0.01（仅有shell，不完整）
1991年10月 发布 0.02（第一个"可用"版本）
1991年12月 发布 0.11（第一个广泛传播的稳定版本）
1994年3月  发布 1.0（正式产品级内核）
```

## 1.3 x86 实模式基础

bootsect.s 和 setup.s 运行在 **实模式（Real Mode）** 下，理解这一点至关重要。

### 实模式内存寻址

实模式下 CPU 使用 **段:偏移** 方式寻址：

```
物理地址 = 段寄存器 × 16 + 偏移量

例：DS=0x9000, BX=0x0200
物理地址 = 0x9000 × 16 + 0x200 = 0x90000 + 0x200 = 0x90200
```

实模式地址空间：**20位 = 1MB (0x00000 ~ 0xFFFFF)**

### 段寄存器（实模式）

| 寄存器 | 名称 | 默认用途 |
|--------|------|---------|
| CS | 代码段 | 当前执行代码的段 |
| DS | 数据段 | 默认数据访问段 |
| ES | 附加段 | 字符串操作目标段 |
| SS | 堆栈段 | 栈操作使用 |

### 通用寄存器（实模式16位）

```
AX (AH:AL) - 累加器，函数调用返回值
BX (BH:BL) - 基址寄存器
CX (CH:CL) - 计数寄存器（loop、rep指令）
DX (DH:DL) - 数据寄存器（I/O端口）
SI          - 源变址（字符串源地址）
DI          - 目标变址（字符串目标地址）
SP          - 栈指针
BP          - 基指针
```

### BIOS 中断服务

Linux 0.11 启动时广泛使用 BIOS 中断：

| 中断号 | 功能 |
|--------|------|
| `int 0x10` | 视频服务 (AH=0x03 读光标, AH=0x13 显示字符串) |
| `int 0x13` | 磁盘服务 (AH=0x02 读扇区, AH=0x08 获取驱动器参数) |
| `int 0x15` | 系统服务 (AH=0x88 获取扩展内存大小) |

## 1.4 x86 保护模式基础

从 setup.s 结束时起，系统进入 **保护模式（Protected Mode）**。

### GDT（全局描述符表）

保护模式不再使用段寄存器直接存放物理地址，而是存放**选择子（Selector）**，指向 GDT 中的描述符。

```
段选择子 (16位)
┌───────────────┬───┬─────┐
│   索引 (13位) │ TI│ RPL │
└───────────────┴───┴─────┘
                 │     └─ 请求特权级 (0=内核, 3=用户)
                 └─ 表指示位 (0=GDT, 1=LDT)

段描述符 (64位)
┌──────────┬────┬────┬──────────┬──────────┐
│ Base[31:24]│ G │ 其他 │ Base[23:0] │ Limit    │
└──────────┴────┴────┴──────────┴──────────┘
```

Linux 0.11 的 GDT 布局：

```
GDT[0]  = 空描述符 (NULL)
GDT[1]  = 内核代码段 (0x00000000, 16MB)
GDT[2]  = 内核数据段 (0x00000000, 16MB)
GDT[3]  = 临时（不使用）
GDT[4]  = 进程0的TSS
GDT[5]  = 进程0的LDT
GDT[6]  = 进程1的TSS
GDT[7]  = 进程1的LDT
...（每个进程占2个GDT条目）
```

### 分页机制

保护模式下 Linux 0.11 使用**二级分页**：

```
线性地址 (32位)
┌──────────┬──────────┬────────────┐
│ 目录索引  │ 页表索引  │  页内偏移  │
│  (10位)  │  (10位)  │   (12位)   │
└──────────┴──────────┴────────────┘
      │           │
      │           └──▶ 页表项 ──▶ 物理页帧地址
      └──────────────▶ 页目录项 ──▶ 页表物理地址
```

## 1.5 内核的关键数据结构预览

在深入代码前，先了解三个最重要的数据结构：

### task_struct —— 进程描述符

```c
// include/linux/sched.h
struct task_struct {
    long state;          // 进程状态: 0=运行, 1=可中断睡眠, 2=不可中断, 3=僵尸, 4=停止
    long counter;        // 时间片剩余(调度用)
    long priority;       // 优先级
    long signal;         // 信号位图
    // ... 信号处理函数
    long blocked;        // 信号屏蔽位图

    int exit_code;       // 退出码
    unsigned long start_code, end_code, end_data, brk, start_stack; // 内存布局
    long pid, father, pgrp, session, leader;  // 进程关系
    unsigned short uid, euid, suid;           // 用户ID
    unsigned short gid, egid, sgid;           // 组ID

    int tty;             // 终端号
    struct m_inode *pwd; // 当前工作目录
    struct m_inode *root;// 根目录
    struct file *filp[NR_OPEN]; // 打开的文件

    struct desc_struct ldt[3]; // 局部描述符表(LDT)
    struct tss_struct tss;     // 任务状态段(TSS)
};
```

### m_inode —— 内存中的inode

```c
// include/linux/fs.h  (内存版inode)
struct m_inode {
    unsigned short i_mode;  // 文件类型和权限
    unsigned short i_uid;   // 属主
    unsigned long  i_size;  // 文件大小(字节)
    unsigned long  i_mtime; // 修改时间
    unsigned char  i_gid;   // 组ID
    unsigned char  i_nlinks;// 硬链接数
    unsigned short i_zone[9];// 数据块号: [0-6]直接块,[7]一级间接,[8]二级间接
    // ... 内存专用字段
    unsigned short i_dev;   // 所在设备
    unsigned short i_num;   // inode编号
    unsigned short i_count; // 引用计数
    // ...
};
```

### buffer_head —— 缓冲区头

```c
// include/linux/fs.h
struct buffer_head {
    char * b_data;           // 缓冲区数据指针(1024字节)
    unsigned long b_blocknr; // 块号
    unsigned short b_dev;    // 设备号
    unsigned char b_uptodate;// 数据是否最新
    unsigned char b_dirt;    // 是否已修改(脏)
    unsigned char b_count;   // 使用计数
    unsigned char b_lock;    // 是否加锁
    struct task_struct * b_wait; // 等待队列
    struct buffer_head * b_prev; // LRU链表
    struct buffer_head * b_next;
    struct buffer_head * b_prev_free;
    struct buffer_head * b_next_free;
};
```

## 1.6 系统启动完整流程图

```
1. BIOS 自检 (POST)
   └─ 从启动盘第一扇区(512字节)加载到 0x7C00

2. bootsect.s 运行 (实模式, 0x7C00)
   ├─ 把自己从 0x7C00 搬到 0x90000
   ├─ 用 int 0x13 加载 setup.s 到 0x90200
   ├─ 用 int 0x13 加载 system 到 0x10000
   └─ 跳转到 0x90200 执行 setup.s

3. setup.s 运行 (实模式, 0x90200)
   ├─ int 0x10/0x15/0x13 收集硬件信息存到 0x90000
   ├─ 将 system 从 0x10000 移动到 0x00000
   ├─ 加载临时 GDT，开启 A20 地址线
   ├─ 重新编程 8259A 中断控制器
   ├─ 设置 CR0.PE=1 进入保护模式
   └─ 远跳 jmpi 0,8 → head.s

4. head.s 运行 (保护模式, 0x00000)
   ├─ 设置段寄存器（使用 GDT 段选择子）
   ├─ 建立初始页目录和4张页表（恒等映射前16MB）
   ├─ 开启 CR0.PG=1 启用分页
   ├─ 设置 IDT（256个门，均指向 ignore_int）
   └─ call _main → init/main.c

5. main() 运行 (内核空间)
   ├─ mem_init()      内存管理初始化
   ├─ trap_init()     设置异常/中断处理程序
   ├─ blk_dev_init()  块设备初始化
   ├─ chr_dev_init()  字符设备初始化
   ├─ tty_init()      TTY初始化
   ├─ time_init()     读取CMOS时间
   ├─ sched_init()    调度器初始化
   ├─ buffer_init()   缓冲区初始化
   ├─ hd_init()       硬盘初始化
   ├─ floppy_init()   软盘初始化
   ├─ sti()           开启中断
   ├─ move_to_user_mode() 切换到用户模式(进程0)
   └─ fork() → init() 创建进程1，执行初始化

6. init() (进程1)
   ├─ setup() 挂载文件系统
   ├─ open("/dev/tty0") 打开终端
   └─ execve("/bin/sh") 启动shell
```
