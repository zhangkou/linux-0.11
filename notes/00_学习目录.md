# Linux 0.11 内核源码学习笔记 — 总目录

> Linus Torvalds, 1991年。约 10,000 行代码，完整实现了一个 Unix-like 操作系统内核。

## 学习路线总览

```
电源上电
    │
    ▼
[BIOS] ──加载──▶ bootsect.s   (实模式, 0x7C00)
                    │
                    │ 自我搬移到 0x90000，加载 setup.s 和 system
                    ▼
               setup.s         (实模式, 0x90200)
                    │
                    │ 收集硬件信息，开启A20，进入保护模式
                    ▼
               head.s          (保护模式, 0x00000)
                    │
                    │ 建立页表，开启分页，跳转 main()
                    ▼
               init/main.c     (C语言内核入口)
                    │
          ┌─────────┼──────────────┬────────────┐
          ▼         ▼              ▼            ▼
       内存管理   进程管理      文件系统     设备驱动
       mm/       kernel/       fs/          kernel/blk_drv/
                                            kernel/chr_drv/
```

## 阅读前置知识

学习 Linux 0.11 内核源码需要以下基础知识：

### 必需
- **C语言**：指针、位操作、内联汇编(`__asm__`)
- **x86汇编基础**：寄存器(eax/ebx/ecx/edx/esp/ebp/esi/edi)，段寄存器(cs/ds/es/ss)，基本指令
- **操作系统概念**：进程、虚拟内存、文件系统、中断

### 推荐
- **x86体系结构**：实模式与保护模式、GDT/IDT、分页机制、特权级
- **Minix操作系统**：Linux 0.11 的直接前身
- **Unix系统编程**：fork/exec/wait/signal系统调用

## 关键常量与限制

| 常量 | 值 | 含义 |
|------|-----|------|
| `NR_TASKS` | 64 | 最大进程数 |
| `HZ` | 100 | 时钟中断频率(100次/秒) |
| `PAGE_SIZE` | 4096 | 页大小(4KB) |
| `BLOCK_SIZE` | 1024 | 文件系统块大小 |
| `NR_OPEN` | 20 | 每进程最大打开文件数 |
| `NR_INODE` | 32 | inode缓存大小 |
| `NR_BUFFERS` | 动态 | 缓冲区数量(按内存大小) |
| 最大内存 | 16 MB | 内核支持上限 |

## 内存布局（启动完成后）

```
物理地址
0x000000 ┌────────────────────┐
         │  页目录 (pg_dir)    │  4KB
0x001000 ├────────────────────┤
         │  页表 pg0~pg3      │  16KB (4张页表)
0x005000 ├────────────────────┤
         │  内核代码+数据      │
         │  (head.s之后的代码) │
         ├────────────────────┤
         │  缓冲区             │  1~4MB (按内存大小)
         ├────────────────────┤
         │  主内存区           │  供进程使用
0xFFFFFF └────────────────────┘  (最大16MB)

0x90000  ┌────────────────────┐
         │  setup.s收集的      │  (启动时临时使用)
         │  硬件信息           │
0x901FC  │  ROOT_DEV          │
0x90200  │  原bootsect位置     │
         └────────────────────┘
```

## 学习建议

1. **按顺序阅读**：从启动流程开始，理解系统如何从零开始运行
2. **结合注释**：Linus 的注释非常宝贵，包含大量设计思路
3. **关注数据结构**：`task_struct`、`m_inode`、`buffer_head` 是三大核心结构
4. **画图辅助**：内存布局、页表结构、文件系统布局都值得手工画图
5. **对照运行**：可以用 QEMU + GDB 跟踪启动过程
