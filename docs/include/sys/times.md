# sys/times.h 文件详解

`include/sys/times.h` 文件是 Linux 0.11 内核以及遵循 POSIX 标准的系统中用于定义进程时间相关信息和函数的头文件。它主要定义了 `struct tms` 结构体，该结构体用于存储一个进程自身及其已终止子进程所消耗的 CPU 时间，并声明了 `times()` 系统调用函数。

## 核心功能

1.  **定义 `struct tms` 结构体**: 用于存储进程的用户CPU时间、系统CPU时间，以及已终止子进程的用户CPU时间和系统CPU时间。
2.  **声明 `times()` 函数原型**: 该系统调用用于获取调用进程的上述时间信息。

## 数据结构详解

### `struct tms`

这是核心的数据结构，用于保存进程的CPU使用时间统计。

```c
#include <sys/types.h> // 包含了 time_t 的定义

struct tms {
	time_t tms_utime;	/* 调用进程在用户模式下执行所花费的CPU时间 */
	time_t tms_stime;	/* 调用进程在内核模式下执行所花费的CPU时间 (例如执行系统调用) */
	time_t tms_cutime;	/* 所有已终止并被当前进程wait过的子进程，在用户模式下执行所花费的CPU时间总和 */
	time_t tms_cstime;	/* 所有已终止并被当前进程wait过的子进程，在内核模式下执行所花费的CPU时间总和 */
};
```
*   **`tms_utime` (User Time)**: 记录了调用进程自身在用户代码中执行所花费的CPU时间。
*   **`tms_stime` (System Time)**: 记录了调用进程自身在内核代码中执行所花费的CPU时间，例如当进程执行系统调用时，内核代表该进程执行代码所花费的时间。
*   **`tms_cutime` (Child User Time)**: 记录了所有已经被其父进程（即调用 `times()` 的这个进程）使用 `wait()` 或 `waitpid()` 等函数回收了终止状态的子进程，在它们各自的用户模式下执行所花费的CPU时间的总和。
*   **`tms_cstime` (Child System Time)**: 记录了所有已经被其父进程回收了终止状态的子进程，在它们各自的内核模式下执行所花费的CPU时间的总和。

所有这些时间值的单位都是**系统时钟滴答数 (clock ticks)**。在 Linux 0.11 中，通常每秒有 `HZ` (在 `<linux/sched.h>` 中定义，通常为100) 个时钟滴答。要将这些值转换为秒，需要除以 `HZ` (或者更准确地说，除以 `sysconf(_SC_CLK_TCK)` 的返回值，但在0.11中直接用 `HZ` 即可)。

## 函数原型声明

### `extern time_t times(struct tms * tp);`

*   **功能**: 获取调用进程的CPU时间统计信息。
*   **参数**:
    *   `struct tms * tp`: 一个指向用户提供的 `struct tms` 结构体的指针。如果此指针非 `NULL`，则 `times()` 函数会将获取到的时间信息填充到这个结构体中。
*   **返回值**:
    *   成功时，返回自系统启动以来（或某个固定时间点以来，如 Epoch）所经过的**时钟滴答数**。这个返回值本身也是一个 `time_t` 类型的值，可以用于粗略计时。
    *   失败时，通常返回 `-1`，并设置 `errno` (尽管在 Linux 0.11 的内核实现中，`sys_times` 总是返回 `jiffies`，不会失败)。
*   **行为**:
    *   内核会收集当前进程的 `utime` 和 `stime`。
    *   内核还会累加当前进程所有已 `wait()` 过的子进程的 `utime` 和 `stime`，分别作为 `cutime` 和 `cstime`。
    *   这些值会被填充到 `tp` 指向的 `struct tms` 结构中（如果 `tp` 非空）。
    *   函数返回当前的系统“时间”（以时钟滴答为单位，通常是全局变量 `jiffies` 的值）。

## 使用场景

*   **程序性能分析**: 开发者可以使用 `times()` 来测量程序不同部分的CPU时间开销，以找出性能瓶颈。
*   **资源记账**: 系统工具（如 `time` 命令）使用 `times()` 来报告一个命令执行所消耗的用户时间、系统时间和其子进程的时间。
*   **Shell**: Shell 在执行完一个命令后，可能会调用 `times()` 来获取并显示该命令的CPU使用情况。
*   **批处理系统**: 在批处理或作业调度系统中，可能会使用 `times()` 来跟踪每个作业的资源消耗。

## 总结

`include/sys/times.h` 为 Linux 0.11 系统提供了一个标准的接口，用于获取进程及其已终止子进程的CPU时间使用情况。通过 `struct tms` 结构体和 `times()` 系统调用，用户程序可以了解其在用户模式和内核模式下的执行时间，这对于性能分析和资源管理非常有用。时间的单位是系统时钟滴答，需要转换为更易读的单位（如秒）时，应考虑系统的 `HZ` 值。
